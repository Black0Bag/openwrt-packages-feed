## ⚠️ Critical Rules

### 1. Sync Mechanism
- `sync.py` runs automatically every day (via cron/subagent)
- It uses **SHA-256 incremental comparison** — only changed files are updated
- It performs a **one-way pull** from upstream repos
- **Files not in upstream will be DELETED** unless added to `protected` set in `sync.py`

### 2. Protected Files (won't be deleted by sync)
The following are safe from deletion:
- `.github/`, `.git/`, `.gitignore`
- `sync.py`, `sync.log`
- `README.md`, `LICENSE`
- `.AGENTS.md` (this file)

**To protect additional files/directories**, edit `sync.py` line containing `protected = { ... }` and add your directory name.

### 3. What Happens When You Modify Files
| Action | Upstream unchanged | Upstream changed |
|--------|-------------------|------------------|
| Modify existing file | ✅ Your change kept | ❌ Overwritten by upstream |
| Add new file | ❌ Deleted next sync | ❌ Deleted next sync |

### 4. Safe Customization Strategies
- **Create patches**: Instead of modifying synced files directly, write patch files in `.patches/` directory and apply them after sync
- **Use protected dirs**: Add custom directories to `protected` set if they contain agent-created files
- **Document changes**: Always comment your modifications with `# AGENT-CUSTOM: <reason>` so you can track them

3. Understand which files are synced vs local-only

### Adding Custom Packages
1. Create a new directory under the appropriate package folder
2. Add it to `protected` in `sync.py` if needed
3. Document the addition with a comment

## 📝 Communication Protocol

When making changes, always include:
1. **Reason**: Why the change was made
2. **Source**: If based on upstream, reference the commit/PR
3. **Impact**: Which files were modified and why

Example commit message format:
```
feat(luci-app-passwall): update xray config parser

- Reason: Fix issue #123 in upstream
- Source: Jonnyan404/cloud-clipboard-go@abc123
- Impact: Modified luasrc/passwall/util_xray.lua
```
